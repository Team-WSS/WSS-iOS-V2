# Learn — 문서에 교훈 누적 (WSS-iOS-V2)

방금 작업에서 얻은 교훈/함정을 문서에 누적한다. 목적은 **문서를 살아있게** 유지하는 것.

기록할 내용: 스킬 인자로 전달된 설명이 있으면 그것을 쓴다(없으면 아래 1번 절차로 후보를 제시해 확인받는다).

## 절차

1. **무엇을 배웠는지 한 줄로 정리.** 코드만 봐도 아는 사실은 적지 말 것 — "코드만 봐선 모르는 것"(숨은 의존, 함정, 예외, 왜)만.
   - 전달된 설명이 비어 있으면, 직전 대화/작업에서 막혔거나 의외였던 지점을 후보로 제시하고 확인받아라.

2. **어디에 적을지 결정** (가장 좁은 범위 우선):
   - 특정 모듈 한정 → `Projects/<Layer>/<Module>/CLAUDE.md`
   - 레이어 전반 → `Projects/<Layer>/CLAUDE.md`
   - 교차 관심사 → 해당 전용 문서 (`docs/TESTING.md`, `docs/CONVENTIONS.md`, `docs/GLOSSARY.md`, `docs/WORKFLOW.md`)
   - 프로젝트 전역 규칙 → 루트 `CLAUDE.md`

3. **해당 문서의 "주의사항 (작업 중 발견 시 누적)" 절에 한 줄 추가.** 절이 없으면 만든다. 중복이면 기존 항목을 보강한다.

4. 무엇을 어디에 적었는지 한 줄로 보고. (커밋은 사용자가 지시할 때만)

## 원칙

- 간결하게. 한 항목 = 한 교훈.
- 코드가 진실. 문서가 코드와 어긋나 있었다면 그 줄도 함께 고친다.
